Sidebar Only Layout

Layout Mode: Sidebar Only
Toggle Breakpoint: Large (992px)
How This Layout Works

In sidebar-only mode, the navbar is hidden when the sidebar is in-flow. The navbar only appears below the breakpoint when the sidebar becomes an offcanvas overlay.

  • Above 992px (lg breakpoint):
    • Sidebar is in-flow (part of the page layout)
    • Sidebar has a fixed width of 260px
    • Navbar is completely hidden
    • Content starts immediately after the sidebar
  • Below 992px:
    • Sidebar becomes an offcanvas overlay
    • Navbar becomes visible
    • Hamburger menu button appears in navbar
    • Clicking hamburger opens sidebar as offcanvas
    • Content takes full width when sidebar is closed
Implementation

This layout uses two body classes:

<body class="layout-sidebar hide-sidebar-lg">
  • layout-sidebar - Hides navbar above breakpoint, shows it below
  • hide-sidebar-lg - Sets breakpoint at 992px for sidebar in-flow vs offcanvas

Key Differences from Navbar Mode:

  • Navbar is hidden above breakpoint (only sidebar visible)
  • Navbar appears below breakpoint (when sidebar is offcanvas)
  • Sidebar is in-flow on desktop (not offcanvas)
  • Sidebar takes up fixed 260px width above breakpoint
  • Provides maximum screen space for content on desktop

Use Case: This layout maximizes screen space on desktop by removing the navbar entirely. Perfect for applications where the sidebar provides all necessary navigation (like file managers, email clients, or admin panels with rich sidebar menus). On mobile, the navbar reappears to house the hamburger button for accessing the sidebar.

Breakpoint Options

You can control when the sidebar switches from in-flow to offcanvas using different breakpoint classes:

  • hide-sidebar-sm - Switch at 576px (rarely used)
  • hide-sidebar-md - Switch at 768px (tablets)
  • hide-sidebar-lg - Switch at 992px (default, recommended)
  • hide-sidebar-xl - Switch at 1200px (larger tablets)
  • hide-sidebar-xxl - Switch at 1400px (keep sidebar in-flow longer)
