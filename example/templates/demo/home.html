To test different outer layout configurations: 1. Open tests/settings.py (or your project's settings.py) 2. Locate the PAGE_CONFIG dictionary 3. Modify the configuration: # Example: Sidebar layout mode PAGE_CONFIG = { "brand": {"text": "Demo", "image_light": "logo.svg"}, "sidebar": {"show_at": "lg", "collapsible": True}, # Enable sidebar at 'lg' breakpoint "navbar": {"fixed": False, "border": True}, "actions": [], } # Example: Navbar-only layout mode PAGE_CONFIG = { "brand": {"text": "Demo", "image_light": "logo.svg"}, "sidebar": {"show_at": False}, # Disable sidebar "navbar": {"fixed": True, "breakpoint": "md"}, # Primary menu in navbar "actions": [], } 4. Restart the development server for changes to take effect 5. Navigate between demo views to see how each inner layout adapts to the new outer layout The outer layout (sidebar vs navbar mode) is GLOBAL and affects all pages. About This Layout

This home view demonstrates the simplest layout configuration: full-width content with no sidebars. The content spans the entire available width within the main application area.

Use the navigation menu to explore other layout demonstrations:

  • Primary Sidebar Demo - Shows a left sidebar with main content
  • Secondary Sidebar Demo - Shows a right sidebar with main content
  • Dual Sidebars Demo - Shows both left and right sidebars with center content

The outer layout (navbar vs sidebar mode) is controlled by PAGE_CONFIG settings and affects the application shell. Inner layouts (content area sidebars) are controlled per-template and demonstrate different content organization patterns.

Quick Tips
  • Resize your browser to see responsive behavior
  • Check the navigation menu for more demos
  • Outer layout mode is set via PAGE_CONFIG
  • Inner layouts vary per view template
