Navbar Only Layout

Layout Mode: Navbar Only
Toggle Breakpoint: Large (992px)
How This Layout Works

In navbar-only mode, the sidebar is always rendered as an offcanvas element and is never displayed inline. The navbar is always visible across all screen sizes.

  • Above 992px (lg breakpoint):
    • Navbar menu items (bell, profile) are visible
    • Hamburger menu button is hidden
    • Sidebar is accessible only via dedicated toggle (if implemented)
  • Below 992px:
    • Navbar menu items (bell, profile) are hidden
    • Hamburger menu button is visible
    • Clicking hamburger opens sidebar as offcanvas overlay
Implementation

This layout uses two body classes:

<body class="layout-navbar hide-sidebar-lg">
  • layout-navbar - Makes sidebar always offcanvas (never in-flow)
  • hide-sidebar-lg - Hides menu items and shows hamburger below 992px

Utility Classes:

  • .hide-if-sidebar-hidden - Applied to navbar menu items
  • .show-if-sidebar-hidden - Applied to hamburger button
Try It Out

Resize your browser window to see the responsive behavior:

  1. At desktop width (≥992px), you should see the bell and profile icons in the navbar
  2. Resize to tablet/mobile width (<992px)
  3. Watch the navbar menu items disappear and the hamburger button appear
  4. Click the hamburger to open the sidebar as an offcanvas overlay
Desktop View (≥992px)
  • Full navbar with menu items visible
  • No hamburger button
  • Sidebar hidden (offcanvas only)
  • Clean, uncluttered interface
Mobile View (<992px)
  • Navbar with only branding
  • Hamburger button visible
  • Menu items hidden to save space
  • Sidebar accessible via hamburger
