Both Layout (Sidebar + Navbar)

Layout Mode: Both (Sidebar + Navbar)
Toggle Breakpoint: Large (992px)
How This Layout Works

In both mode, the navbar and sidebar are both visible. All navigation happens through the sidebar. The navbar provides a consistent header for branding, search, notifications, or user controls.

  • Above 992px (lg breakpoint):
    • Sidebar is in-flow (part of the page layout)
    • Sidebar has a fixed width of 260px
    • Navbar is visible with branding and action buttons
    • Navbar menu items (Home/Products/Users) are never shown
    • Hamburger button is hidden (sidebar already visible)
    • All navigation through sidebar
  • Below 992px:
    • Sidebar becomes an offcanvas overlay
    • Navbar remains visible
    • Navbar menu items remain hidden
    • Hamburger button appears in navbar
    • Clicking hamburger opens sidebar as offcanvas
    • Content takes full width when sidebar is closed
Implementation

This layout uses two body classes:

<body class="layout-both hide-sidebar-lg">
  • layout-both - Shows both navbar and sidebar, hides navbar menu
  • hide-sidebar-lg - Sets breakpoint at 992px for responsive behavior

Key Features:

  • Navbar and sidebar both visible on desktop and mobile
  • Navbar menu items always hidden (all navigation via sidebar)
  • Sidebar in-flow on desktop, offcanvas on mobile
  • Hamburger only shows when sidebar is offcanvas
  • Navbar provides space for branding, search, and global actions
Try It Out

Resize your browser window to see the responsive behavior:

  1. At desktop width (≥992px), you should see:
    • Sidebar visible on the left (260px width, in-flow)
    • Navbar visible at the top with branding
    • Action buttons (bell, profile) in navbar
    • NO menu items (Home, Products, Users) in navbar
    • No hamburger button (sidebar already visible)
  2. Resize to tablet/mobile width (<992px) and watch:
    • Sidebar disappears from page flow (becomes offcanvas)
    • Navbar remains visible
    • Navbar still has NO menu items
    • Hamburger button appears in navbar
    • Content takes full width
  3. Click the hamburger to open the sidebar as an offcanvas overlay

Use Case: This layout provides a consistent navbar header across all pages while keeping all navigation in the sidebar. Perfect for applications that need persistent branding, search, notifications, or user controls in a top bar, but want all page/section navigation consolidated in a sidebar. Common in enterprise applications, SaaS platforms, and admin panels.
